# Video Playback Events Topic

This document describes the event stream used to expose video playback telemetry for the StreamForge platform.
The stream is consumed via the `Get_video_playback_events` data product port and implemented using Apache Kafka.

## Purpose

The Video Playback Events topic captures fine-grained playback activity generated by users as they interact with videos.
These events support real-time analytics, monitoring, and downstream enrichment pipelines.

Typical use cases include:
- Measuring video engagement and retention
- Powering real-time dashboards
- Feeding machine-learning pipelines
- Supporting fraud or anomaly detection

## Access Pattern

This topic is exposed as an **event stream**.
Consumers subscribe to the stream and process events incrementally as they are published.

The corresponding data product port supports **Event Stream Access**.

## Event Semantics

Each message represents a playback-related event emitted by the StreamForge player.

Events are immutable and append-only.

Examples of event types include:
- playback_started
- playback_paused
- playback_resumed
- playback_completed
- playback_error
